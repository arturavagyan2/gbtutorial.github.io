GitHub

What is Github?

GitHub is a website and cloud-based service that helps developers store and manage their code, as well as track and control changes to their code.

To understand exactly what GitHub is, you need to know two connected principles:

  • Version Control
  • Git

Version Control

Version control helps developers track and manage code changes to a software project. As a software project grows, version control becomes essential.

What is Git?

Git is a specific open-source version control system created by Linus Torvalds in 2005.

Git is a distributed version control system, meaning that the entire codebase and history are available on every developerâ€™s computer, allowing easy branching and merging.

Git Branching

What is Git Branching

At the start when you create a repository it will be your Master Branch.So when you create a New Branch it will have all the files that was in the Master Branch but when you change anything in the New Branch it will not affect the Master Branch.

When to use branching?

We will use it when we want to test a New Feature but we dont want to it affect the main branch which is master if anything happens. other use is when we want to fix the main(master) brach.

Types of Branching

  • HotFix Branch
  • Feature Branching

GitHub Projects

What are GitHub Projects?

GitHub Projects is a project management tool integrated into GitHub that helps you:

  • Organize tasks visually
  • Track progress of issues and pull requests
  • Collaborate with team members
Think of it like a To-Do List, but smarter!

Key Features of GitHub Projects

  • Kanban Boards: Organize tasks in columns (To Do, In Progress, Done)
  • Issue Integration: Link GitHub Issues & Pull Requests to your project
  • Custom Fields: Add priority, status, or deadlines to tasks
  • Automation: Move tasks automatically based on activity
  • Real-Time Updates: Track progress in sync with your repo
